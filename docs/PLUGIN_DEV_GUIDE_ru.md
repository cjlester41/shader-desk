# Руководство по разработке плагинов

Это руководство поможет вам создать собственный эффект для `interactive-wallpaper`. Архитектура проекта построена по принципу "шейдер — на первом месте": вы пишете GLSL-код, описываете настраиваемые параметры в комментариях, а специальный скрипт автоматически генерирует необходимый C++ и CMake код-обвязку.

Это позволяет сосредоточиться на визуальной части, минимизируя рутинную работу с C++.

## ✨ Основная концепция

1. **Создание директории**: Вы создаете новую папку для своего плагина в директории `plugins/`.
2. **Написание GLSL-шейдера**: Вы пишете фрагментный шейдер. Все настраиваемые параметры (например, цвет, скорость, интенсивность) объявляются в виде `uniform`-переменных и описываются в специальных комментариях.
3. **Генерация кода**: Вы запускаете Python-скрипт
   `generate_plugin.py`, который парсит ваш шейдер и создает:
   - C++ класс эффекта (`.hpp`, `.cpp`), который реализует основной интерфейс `WallpaperEffect`.
   - `CMakeLists.txt` для сборки плагина в виде разделяемой библиотеки (`.so`).
1. **Сборка**: Вы добавляете плагин в общую систему сборки и компилируете проект.
2. **Установка и запуск**: Копируете скомпилированный `.so`-файл и шейдеры в директорию конфигурации, после чего эффект становится доступен в приложении.

## 🛠️ Пример: Создание эффекта "Simple Wave"

Давайте создадим простой эффект анимированных волн, чтобы продемонстрировать весь процесс.

### Шаг 1: Создание структуры папок

Внутри директории `plugins/` создайте новую папку для вашего эффекта. Название должно быть в стиле `kebab-case`.

```bash
cd plugins
mkdir simple-wave-effect
cd simple-wave-effect
mkdir shaders
```

Структура должна выглядеть так:

```
plugins/
├── ico-sphere-effect/
├── pulse-color-effect/
└── simple-wave-effect/   <-- Ваша новая папка
	└── shaders/
```

### Шаг 2: Написание фрагментного шейдера

Создайте файл c вашим шейдером.
Для примера создадим `plugins/simple-wave-effect/shaders/wave_frag.glsl` и добавим в него следующий код:

```GLSL
// plugins/simple-wave-effect/shaders/wave_frag.glsl
#version 300 es
precision mediump float;

// Входные данные от вершинного шейдера (координаты текселя)
in vec2 v_uv;

// Выходной цвет
out vec4 FragColor;

// Стандартные uniform-переменные, всегда доступны
uniform float time;
uniform vec2 resolution;

// Пользовательские параметры, которые будут управляться из конфига.
// Формат: @param <имя_в_коде> | <тип> | <значение_по_умолчанию> | <описание>

// @param wave_color | vec3 | 0.1, 0.5, 1.0 | Основной цвет волн (R,G,B)
uniform vec3 wave_color;

// @param speed | float | 2.5 | Скорость движения волн
uniform float speed;

// @param frequency | float | 20.0 | Частота (количество) волн на экране
uniform float frequency;

// @param is_inverted | bool | false | Инвертировать ли цвета
uniform bool is_inverted;

void main() {
// Нормализуем координаты, чтобы центр был (0,0)
vec2 uv = v_uv - 0.5;

// Простое уравнение волны, зависящее от времени и расстояния от центра
float wave = sin(length(uv) * frequency - time * speed);

// Сделаем волны более плавными
wave = smoothstep(0.0, 1.0, wave);

vec3 final_color = wave_color * wave;

if (is_inverted) {
final_color = vec3(1.0) - final_color;
}

FragColor = vec4(final_color, 1.0);
}
```

**ВАЖНО использовать специальные комментарии `@param` для параметров, которые вы хотите изменять в `config.json`**:
`// @param <имя> | <тип> | <умолчание> | <описание>`

- **имя**: Должно точно совпадать с именем `uniform`-переменной в шейдере.
- **тип**: Поддерживаемые типы: `float`, `int`, `bool`, `vec3`.
- **умолчание**: Значение по умолчанию. Для `vec3` используется формат `R, G, B`.
- **описание**: Текстовое описание, которое будет использоваться в утилитах.

Вам также понадобится **вершинный шейдер**.
Создайте файл `plugins/simple-wave-effect/shaders/wave_vert.glsl`:

```
// plugins/simple-wave-effect/shaders/wave_vert.glsl
#version 300 es

// Выходные данные для фрагментного шейдера
out vec2 v_uv;

// Массив вершин для полноэкранного треугольника
const vec2 positions[3] = vec2[](
vec2(-1.0, -1.0),
vec2(3.0, -1.0),
vec2(-1.0, 3.0)
);

void main() {
// Выбираем вершину
vec2 pos = positions[gl_VertexID];

// Передаем UV-координаты во фрагментный шейдер
// (0,0) в нижнем левом углу, (1,1) в верхнем правом
v_uv = pos * 0.5 + 0.5;

// Устанавливаем позицию вершины
gl_Position = vec4(pos, 0.0, 1.0);
}
```

### Шаг 3: Генерация C++ кода

Теперь, когда шейдеры готовы, вернитесь в корневую директорию проекта и запустите скрипт `generate_plugin.py`. Укажите ему путь к папке вашего плагина.

```
# Находясь в корневой папке проекта (shader-desk/)
python3 plugins/generate_plugin.py plugins/simple-wave-effect
```

Скрипт проанализирует `@param` в вашем `.glsl` файле и сгенерирует три файла внутри `plugins/simple-wave-effect/`:

- `simple-wave-effect.hpp` (Заголовочный файл класса)
- `simple-wave-effect.cpp` (Файл реализации)
- `CMakeLists.txt` (Файл для сборки)

### Шаг 4: Сборка и установка плагина

**Скомпилируйте проект**:
```
cd build
cmake ..
make 
```

После успешной сборки в директории `build/effects/` появятся:
- `simple-wave-effect.so` — ваш плагин.
- папка `shaders/simple-wave-effect/` с вашими шейдерами.

**"Установите" плагин**: Приложение ищет плагины в `~/.config/interactive-wallpaper/effects`.

Скопируйте или создайте символическую ссылку на результаты сборки.

### Шаг 6: Активация и настройка нового эффекта

1. **Обновите конфигурацию**: Запустите `./src/config-manager.sh init`, чтобы он нашел ваш новый плагин и добавил его параметры в `config.json` .
2. **Посмотрите список доступных эффектов**:

```
./src/config-manager.sh list
# Вывод должен содержать:
# - Ico Sphere Effect
# - Pulse Color Effect
# - Simple Wave Effect  <-- Ваш новый эффект!
```

3. **Переключитесь на новый эффект**:
   `./src/config-manager.sh switch "Simple Wave Effect"`

4. **Запустите/перезапустите обои**:
   `./run.sh restart`

Теперь на вашем рабочем столе должны появиться синие волны! Вы можете настраивать их параметры "на лету":

```
# Изменить цвет на огненный
./src/config-manager.sh set wave_color '[1.0, 0.2, 0.0]'

# Ускорить волны
./src/config-manager.sh set speed 5.0

# Инвертировать цвета
./src/config-manager.sh set is_inverted true
```

Приложение автоматически подхватит изменения в `config.json` и применит их.

Поздравляем! Вы создали свой первый плагин. Теперь вы можете экспериментировать с более сложными шейдерами и параметрами.
